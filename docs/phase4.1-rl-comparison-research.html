The Three Questions

QuestionAnswer
How to design the experiments? Hold base model, SFT init, dataset subset, reward function, reference model, generation HPs, evaluation protocol, and seeds constant. Vary only the RL loss / advantage computation. Run ≥ 3 seeds per algorithm on the same 2 K subset used in Phase 4. Report Interquartile Mean (IQM) with bootstrap 95% CIs. Anchor everything to GRPO — the table is always "GRPO vs X," not a free-for-all.
How to keep conditions consistent? Extend the Phase 4 grpo_step() into a dispatcher over swappable loss modules that share generation, log-prob computation, reference-model handling (PEFT disable_adapter_layers()), reward computation, and KL coefficient. Keep ε, β, lr, temperature, G, batch size, gradient accumulation, optimizer, and sample budget identical across algorithms. Only the advantage/loss math changes. Tune learning rate per algorithm on a 3-point grid — it is the one HP where "same value" is unfair.
Which algorithms? Four on-policy algorithms that all fit in T4 VRAM, all use a scalar-per-sequence reward, all reuse the existing generation loop, and collectively cover the family tree of critic-free RLHF: Dr. GRPO, RLOO, REINFORCE++, and PPO with group-mean baseline.

Algorithm Shortlist

Four algorithms were selected to satisfy four constraints simultaneously: applicable to sequence-level scalar reward on decoder-only ASR, fits T4 VRAM at G=2, implementable as a ~50-line LossModule, and scientifically interesting.

1 Dr. GRPO
Liu et al. 2025		 r − group_mean (no std, no length norm) Clipped ratio + β·KL Group-relative Reference
2 RLOO

4 PPO (group-mean)
Schulman 2017, Huang 2022		 Same as Dr. GRPO — the key control PPO clip + β·KL + optional entropy bonus Clipped ratio Challenger

Why these four, and what each comparison isolates

  • Dr. GRPO vs PPO (group-mean): Identical advantages, different loss machinery. Isolates what the PPO clip + entropy bonus buy over plain REINFORCE with group-mean baseline.
  • RLOO vs Dr. GRPO: Isolates the effect of the baseline (leave-one-out vs group mean). At G=2, these should be mathematically equivalent up to a constant; observed divergence is a pure implementation-detail finding.
  • REINFORCE++ vs GRPO: Global vs per-prompt normalization on heterogeneous prompt sets — exactly the setting where REINFORCE++'s claim should matter, given our FS/CV demographic split.

Rejected Candidates

AlgorithmVerdictWhy rejected
PPO with separate critic Out of scope Doubles VRAM (adds a second model head). Explicitly out of scope per REQUIREMENTS.md. Not "fair" since every other candidate is critic-free.
DPO / IPO / SimPO / ORPO Wrong shape Require converting scalar composite reward to pairwise preferences — information-lossy. No audio trainer in TRL. Would need a whole second data pipeline for one algorithm.
KTO Experimental Requires binary desirable/undesirable labels, not scalar rewards. Marked experimental in TRL v1.0. No audio support.
RAFT / Rejection sampling Duplicates Phase 6 Already planned as Phase 6 ("Rejection Sampling — Fairness-Aware SFT"). Including here would duplicate work.
ReMax Marginal At G=2, functionally nearly indistinguishable from RLOO with the greedy sample as baseline. Marginal value over RLOO.
DAPO (full) Too many knobs Asymmetric clip + dynamic sampling + token-level loss — each is a separate ablation. Deferred unless entropy collapse is observed.
GSPO Wrong regime Full-sequence importance ratio helps long sequences; ASR transcriptions are < 128 tokens. Marginal value.
TRL PPOTrainer (v1) Deprecated Legacy v1 RLHF loop is being deprecated; never compatible with non-causal-LM models.

Sample budget equity

Every algorithm sees the same counts per seed:

  • Rollouts: 200 × 1 × 2 = 400 environment interactions
  • Reward queries: 400 RewardComputer calls
  • Gradient updates: 200 optimizer steps (with grad_accum=4)
  • Ref-model forward passes: 400 teacher-forcing calls via disable_adapter_layers()

Aggregation and statistics

  1. Report Interquartile Mean (IQM) with bootstrap 95% CI across seeds (Agarwal 2021).
  2. Also report arithmetic mean ± SE for readers who want the classic view.
  3. Learning-curve plot with shaded band = IQM ± 95% bootstrap CI.
  4. Pairwise bootstrap (10 000 resamples) of Δ = mean(challenger) − mean(GRPO).
  5. Per Henderson 2018, also report Welch t-test and Mann–Whitney U alongside the bootstrap.
Honest caveat: With N=3 seeds, no pairwise test will reach p < 0.05 unless the effect is very large. The value of the table is in showing the direction of differences and the overlap of CIs. Do not sell underpowered tests as conclusive.

Plots produced at phase end

  • Reward learning curves — one panel, four lines, shaded bootstrap CI band.
  • Pareto scatter — (WER, gap) per (algorithm, seed) with per-algorithm 95% CI ellipse.
  • Wall-clock efficiency — bar chart: steps-to-90%-of-final-reward per algorithm.
  • VRAM bar chart — peak VRAM per algorithm (should be nearly identical).

Reward-hacking gate

CheckTrigger
wer_floor_breachFinal mean WER > baseline × 1.2
length_explosionFinal mean completion length > 2× initial

If any check fires for any run, that run is marked DEGRADED and excluded from the "winner" decision, but still reported.

Top 10 Pitfalls

#PitfallMitigation
1 Per-algo KL coefficient differences silently pick the winner Hold β = 0.04 identical; log both raw kl and β·kl
2 Shared lr is unfair — different losses have different gradient magnitudes Per-algo lr grid on 50-step probe with seed=42 only
3 Single-seed lr winners are noise artifacts Flag for re-probe if chosen lr differs by > 1 OOM from grid center
4 Phase 4's unique-answer problem inherited — all algos fail equally Preflight check frac_zero_std < 0.5 before launch
5 Non-deterministic adapter load across seeds Hash state_dict, use torch.use_deterministic_algorithms(True)
6 Length bias disguised as "PPO is better" Dr. GRPO length normalization across all four algorithms
7 Entropy collapse from symmetric ε=0.2 clip Log entropy every step; mark DEGRADED if < 0.3× initial
8 Refactor confounder: Dr. GRPO module diverges from Phase 4 Regression test — Dr. GRPO reward curve must match Phase 4 within 0.5% at every logged step
9 Silent NaN step-skips create sample-budget mismatch Log n_nan_skipped; mark DEGRADED if > 5
10 Reporting single-number winners from 3 seeds Every numeric claim accompanied by 95% bootstrap CI and CI-overlap sentence

Methodological note

Novel contribution: To the researcher's knowledge, fair head-to-head RL algorithm comparison on ASR fairness has not been published. Phase 04.1 is a direct application of the 2024–2026 RLHF comparison methodology (Henderson 2018 + Agarwal 2021 + Ahmadian 2024) to the audio/ASR setting, with Dr. GRPO as the anchor.
