CORE INSIGHT
"SFT Expands, RL Squeezes"
A 2025 paper (arXiv:2509.21128) provides the clearest explanation of why alternating SFT and RL is the modern standard.

SFT Supervised Fine-Tuning

Expands the space of correct behaviors. Teaches new solution strategies by imitation — directly showing the model what good outputs look like.

  • Rapid initial learning from demonstrations
  • Establishes format, readability, structure
  • Provides warm start (avoids wasting RL compute)
  • Teaches domain-specific patterns (transcription format)

RL Reinforcement Learning

Compresses incorrect behaviors. Eliminates bad outputs through negative signal, discovering solutions no human demonstrated.

  • Discovers novel solutions not in training data
  • Generalizes better (SFT tends to memorize)
  • Higher asymptotic performance ceiling
  • Can optimize non-differentiable objectives (fairness gaps)
THE STANDARD PIPELINE
4-Stage Alternating SFT ↔ RL
DeepSeek-R1, Qwen3, and Llama all converged on this multi-round pattern. Each stage builds on what the previous one accomplished.
Stage 1 — SFT
Cold-Start Supervised Fine-Tuning
Teach the model correct output format and domain basics through imitation. Establishes readable structure, prevents language mixing, provides a competent starting point for RL.
Stage 2 — RL (GRPO)
Reasoning-Oriented Reinforcement Learning
Large-scale RL on task-specific objectives with rule-based rewards. The model discovers novel solutions and eliminates incorrect behaviors that SFT couldn't suppress.
Stage 3 — Rejection Sampling + SFT
Bridge: Capture RL Discoveries as SFT Data
Sample 16-64 responses from the RL checkpoint, keep only correct/high-quality ones. Use filtered responses as new SFT training data. DeepSeek-R1 produced ~600k samples this way. Retrain from base model on this higher-quality dataset.
Stage 4 — RL
General Reinforcement Learning
Second RL phase targeting broader objectives: helpfulness, harmlessness, instruction-following. Uses both rule-based rewards and reward models for open-ended tasks.

WHY ALTERNATE
Rejection Sampling is the Bridge
The key mechanism that connects RL rounds back to SFT rounds, creating a virtuous improvement cycle.
1

RL Discovers

Reinforcement learning finds novel, high-quality solutions that were never in the original training data. It explores policy space guided by reward signals.

2

Rejection Sampling Captures

Generate 16-64 candidate outputs per prompt from the RL model. Filter to keep only correct/high-quality responses. This distills RL discoveries into clean supervised examples.

3

SFT Consolidates

Retrain on the filtered, higher-quality data. The next SFT round has strictly better training data than the previous round. The cycle repeats.

The Virtuous Cycle

RL discovers better solutions → rejection sampling captures them as supervised data → next SFT round trains on higher-quality data → better starting point for next RL round → repeat

PRACTICAL: LORA
LoRA Without Regret
TRL's guide for multi-stage LoRA training. The key insight: SFT needs high rank, RL needs low rank.
SFT

High Rank: r = 256

SFT needs capacity to learn from diverse training data. High rank captures the breadth of transcription patterns.

RL

Low Rank: r = 1–32

Policy gradient extracts ~1 bit of information per episode. Minimal parameter capacity is sufficient.

MERGE

Merge Between Stages

Merge SFT LoRA into base model, then apply fresh LoRA for RL. Never stack adapters — causes training instability.

Why target_modules="all-linear"?

Applying LoRA to all weight matrices is critical. Increasing rank does NOT compensate for targeting fewer layers. Always use all-linear, adjust rank instead.

Why not stack LoRA adapters?

Stacking (LoRA on top of LoRA) causes gradient interference and training instability. Merging the first adapter into the base weights gives a clean starting point for the next stage.

What about catastrophic forgetting between stages?

The BRIDGE method (arXiv:2509.06948) proposes separate LoRA for SFT and base params for RL, with gradient fusion. Achieves 44% faster training with 13% gain on Qwen2.5-3B. MIT 2025 study also shows RL minimizes forgetting vs SFT.

FOR OUR PROJECT
Proposed Training Pipeline
Applying the alternating pattern to Qwen3-ASR-1.7B fairness fine-tuning on a single RTX A4000 (16GB).
Phase 1 — SFT Baseline
Standard LoRA SFT (r=256)
Fine-tune Qwen3-ASR decoder on Fair-Speech + CommonVoice transcription data. Standard cross-entropy loss. This produces the "standard LoRA baseline" (λ=0) AND the starting point for GRPO. Merge LoRA into base when done.
Phase 2 — GRPO
Fairness-Aware GRPO (fresh LoRA, r=16)
Apply fresh LoRA on merged SFT model. Composite reward: R = (1-λ)(1-WER) + λ(-|WER_g - WER_mean|). Sweep λ to trace the accuracy-fairness Pareto frontier.
Phase 3 — Optional: Rejection Sampling
Capture Fairness-Aware Transcriptions
If time permits: sample from best GRPO checkpoint, filter for low-gap + high-accuracy transcriptions, use as data for a second SFT round. Creates a model that is both fair and accurate without RL overhead at inference.
Phase 4 — Baselines
FairLoRA + Group-DRO + ICASSP Comparisons
Train the three comparison baselines: FairLoRA (supervised fairness regularizer), Group-DRO (distributionally robust optimization), and ICASSP 2026 fairness-prompted fine-tuning. All plotted on the same Pareto frontier.

FAQ
Common Questions
Answers to questions about the SFT+RL alternating training paradigm.
Can I skip SFT and go straight to GRPO?

Technically yes — DeepSeek-R1-Zero proved it. But the model suffers from poor formatting, endless repetition, and language mixing. The cold-start SFT is cheap and dramatically improves RL stability. Don't skip it.

Can I skip RL and just do SFT?

You can, but performance plateaus. SFT can't optimize for non-differentiable objectives like demographic fairness gaps. It also can't suppress incorrect behaviors as effectively — it "preserves incorrect trajectories with non-negligible probability mass."

Why does DeepSeek retrain from the base model in Stage 3?

Retraining from base with higher-quality data avoids accumulating training artifacts from the RL stage. The rejection-sampled data already encodes the RL improvements, so you get the benefits of RL without the formatting issues.

Is DPO a form of RL?

Yes, DPO (Direct Preference Optimization) is an offline RL algorithm that optimizes the same objective as RLHF but without explicitly training a reward model. Llama 3.1 preferred it over PPO for stability reasons. It fits the same alternating pattern.

How many alternating rounds are needed?

Current practice: 2 rounds (SFT→RL→SFT→RL). More rounds show diminishing returns. For a class project with limited compute, even 1 round (SFT→GRPO) captures most of the benefit.

What does TRL support for multi-stage training?
TRL v1.0 (April 2026) provides SFTTrainer, GRPOTrainer, DPOTrainer, and RewardTrainer. Multi-stage is implemented as sequential script runs, not a single loop. Each trainer accepts a peft_config for LoRA integration.
