Overview

This document catalogues all evaluation metrics relevant to ASR fine-tuning with LoRA, organized by when and why each metric is used. Context: Optuna HP search on a 500-sample subset, 100-step prototype runs, 50-sample eval set, targeting Whisper / wav2vec2 / Qwen3-ASR.

How to read this document. Each metric includes its formula, what it measures, when to use it, and a verdict badge indicating relevance to our specific setup: Primary = use this, Useful = secondary signal, Niche = situational, Not applicable = doesn't fit our context.

HP Search Metrics Prototype Phase

Metrics for ranking hyperparameter configurations during short prototype runs. The goal is to identify promising configs cheaply, not to measure final model quality.

MetricFormulaMeasuresVerdict
Eval Cross-Entropy Loss $-\frac{1}{N}\sum_{i=1}^{N}\log p(y_i \mid x_i)$ Log-probability of the correct token at each position, averaged Primary
Perplexity $\exp(\text{CE Loss})$ Exponentiated cross-entropy — more interpretable scale Niche
AULC $\int_0^T \text{eval\_loss}(t)\, dt$ Area under the learning curve — captures convergence speed + final value Primary
Top-k Accuracy $\frac{1}{N}\sum \mathbb{1}[y_i \in \text{top-}k(p_i)]$ Whether the correct token is in the model's top $k$ predictions Niche
Token-level Accuracy $\frac{1}{N}\sum \mathbb{1}[\hat{y}_i = y_i]$ Exact match at each position Not applicable

Area Under the Learning Curve (AULC)

Recommended for HP search tie-breaking

Instead of comparing configs by their final eval loss at step 100 (a single noisy point), integrate the entire loss trajectory via trapezoidal approximation:

$$\text{AULC} = \sum_{i=1}^{n-1} \frac{L(t_i) + L(t_{i+1})}{2} \cdot (t_{i+1} - t_i)$$

Why it helps: Your top 5 trials span only 0.159–0.175 eval loss — within noise range for $n=50$. AULC aggregates across all evaluation checkpoints, effectively increasing signal. It also penalizes instability (oscillating loss curves accumulate more area) and rewards fast convergence.

Bottom line: Use eval loss as the primary Optuna objective and AULC to break ties among top configs. Perplexity is a monotonic transform of loss (same ranking). Token-level accuracy is too coarse for 300 tokens.

ASR Quality Metrics Full-Scale Eval

Metrics for measuring transcription quality of the final trained model. These require running full decoding, so they're too expensive for HP search but essential for final evaluation.

Primary: Edit-Distance Metrics

MetricFormulaMeasuresVerdict
WER (Word Error Rate) $\frac{S + D + I}{N}$ Word-level edit distance between prediction and reference Primary
CER (Character Error Rate) $\frac{S_c + D_c + I_c}{C}$ Character-level edit distance — finer-grained than WER Useful
SER (Sentence Error Rate) $\frac{\text{sentences with any error}}{\text{total sentences}}$ Proportion of utterances with any transcription error Useful
MER (Match Error Rate) $\frac{S + D + I}{S + D + C}$ Normalized for alignment length; $C$ = correct matches Niche
WIL (Word Info Lost) $1 - \frac{C^2}{N \cdot N'}$ Penalizes both substitutions and deletions; $N'$ = hypothesis length Niche
WIP (Word Info Preserved) $\frac{C^2}{N \cdot N'}$ $1 - \text{WIL}$; sometimes easier to interpret Niche

Where $S$ = substitutions, $D$ = deletions, $I$ = insertions, $N$ = reference word count, $C$ = correct matches.

WER is the gold standard for ASR. CER is a useful secondary signal, especially when WER is noisy or for languages with agglutinative morphology. The others are niche — use if your paper needs comprehensive coverage.

WER Decomposition: Sub / Del / Ins Rates

WER decomposes into three error types, each revealing different failure modes:

ComponentFormulaWhat it reveals
Substitution Rate $S / N$ Model hears the word but predicts a different one (confusion)
Deletion Rate $D / N$ Model misses words entirely (under-generation, silence handling)
Insertion Rate $I / N$ Model hallucinates extra words (over-generation)
Fairness insight: If one demographic group has disproportionately high deletions, the model is systematically dropping their words — a more pointed finding than uniform WER increase. This decomposition is worth reporting per-group.

How Edit Types Are Determined: DP Alignment

WER uses minimum edit distance (Levenshtein distance) via dynamic programming to align the reference and hypothesis word sequences. The algorithm builds a cost matrix $d[i][j]$ representing the minimum edits needed to transform the first $i$ hypothesis words into the first $j$ reference words. Each cell is computed from three possible moves:

MoveDirection in MatrixError TypeMeaning
Diagonal (words match) $d[i{-}1][j{-}1] + 0$ Correct Words match — no error, zero cost
Diagonal (words differ) $d[i{-}1][j{-}1] + 1$ Substitution Model said the wrong word
Move right (skip ref word) $d[i][j{-}1] + 1$ Deletion Model dropped a word that should be there
Move down (skip hyp word) $d[i{-}1][j] + 1$ Insertion Model hallucinated an extra word

After filling the matrix, a backtrace from $d[M][N]$ to $d[0][0]$ deterministically assigns each position to exactly one operation. There is no ambiguity — the minimum-cost path defines the alignment.

Secondary: N-gram and Semantic Metrics

MetricWhat it measuresVerdict
Entity WER WER computed only on named entities (names, numbers, places) Useful
BERTScore Semantic similarity via contextual embeddings — captures "close enough" transcriptions that WER penalizes Niche
BLEU N-gram precision with brevity penalty — borrowed from MT Niche
METEOR N-gram matching with stemming and synonyms — more recall-oriented Niche
TER (Translation Edit Rate) Like WER but allows phrase shifts as single edits Niche
Entity WER is the one addition worth considering for your fairness paper. If the model fails on names from certain demographics, that's a concrete, citable finding. BLEU/METEOR/TER are from machine translation and rarely used in ASR evaluation.

Fairness Metrics Analysis Phase

Metrics for quantifying and testing performance gaps across demographic subgroups.

MetricFormulaMeasuresVerdict
WER Gap $\max_{g \in G} \text{WER}_g - \min_{g \in G} \text{WER}_g$ Absolute spread between best- and worst-performing groups Primary
WER Ratio (MMR) $\frac{\max_{g \in G} \text{WER}_g}{\min_{g \in G} \text{WER}_g}$ Multiplicative disparity — scale-invariant Primary
Per-group WER $\text{WER}_g$ for each $g \in G$ Breakdown by demographic subgroup Primary
Statistical Significance Wilcoxon / bootstrap CI Whether observed group differences are real or noise Primary
Per-group Sub/Del/Ins Error type breakdown per group Whether disparities are driven by specific error types Useful
Entity WER by Group Entity-only WER per group Name/number recognition bias Useful
$$\text{MMR} = \frac{\max_{g \in G}\, \text{WER}_g}{\min_{g \in G}\, \text{WER}_g}$$

An MMR of 1.0 means perfect parity. Higher values indicate greater disparity. For example, MMR = 1.5 means the worst-performing group has 50% higher WER than the best.

Recommended reporting: WER gap + WER ratio + per-group WER table + statistical significance. Add Sub/Del/Ins breakdown if you find non-uniform error type distributions across groups — this strengthens the fairness narrative.

Probabilistic / Calibration Metrics Model Analysis

Metrics for assessing the quality of the model's probability estimates, not just its predictions.

MetricFormulaMeasuresVerdict
NLL per Token $-\frac{1}{N}\sum \log p(y_i \mid x_i)$ Same as cross-entropy loss; standard training objective Primary
Predictive Entropy $H = -\sum_v p(v) \log p(v)$ Model uncertainty — high entropy = model is unsure about output Useful
ECE (Expected Calibration Error) $\sum_{b=1}^{B} \frac{n_b}{N} |\text{acc}_b - \text{conf}_b|$ Gap between confidence and actual accuracy, binned Niche
Brier Score $\frac{1}{N}\sum (p_i - y_i)^2$ Mean squared error of probability estimates Niche
Predictive entropy is the one calibration metric worth checking. If the model has systematically higher entropy on one demographic group's speech, it's less certain about that group — an additional fairness signal beyond WER. ECE and Brier score are informative but rarely reported in ASR papers.

Training Diagnostics Debugging

Metrics for diagnosing training stability and config quality. These are not ranking metrics — they help you understand why a config is good or bad.

MetricWhat it signalsWhen to worry
Loss Curve Shape Convergence quality Still oscillating at step 100 → unstable config
Convergence Speed How quickly loss drops in first 25 steps Slow start often predicts poor final result
Gradient Norms Training stability Spikes early on predict poor final performance
Adapter Weight Norms Solution quality for LoRA specifically High rank + large magnitude → spurious local minima
Effective Rank Utilization of adapter capacity Effective rank $\ll$ nominal rank → capacity is wasted
Train-Eval Gap Overfitting signal Large gap on 500 samples → will worsen at scale

Effective Rank

The effective rank of a LoRA adapter matrix $\Delta W = BA$ measures how much of the allocated rank is actually being used:

$$\text{erank}(\Delta W) = \exp\!\left(-\sum_{i=1}^{r} \bar{\sigma}_i \log \bar{\sigma}_i\right), \quad \bar{\sigma}_i = \frac{\sigma_i}{\sum_j \sigma_j}$$

Where $\sigma_i$ are the singular values of $\Delta W$. If $\text{erank} \approx r$, the adapter is using full capacity. If $\text{erank} \ll r$, the adapter has collapsed to a lower-dimensional solution and the extra rank is wasted.

For your HP search: Your finding that rank=4 dominated the top trials is consistent with small datasets needing low rank. Checking effective rank on a rank=8 or rank=16 trial would likely show $\text{erank} \approx 3\text{-}5$, confirming rank=4 is the right choice.

Efficiency / Deployment Metrics Inference

MetricFormulaMeasuresVerdict
RTF (Real-Time Factor) $\frac{\text{inference time}}{\text{audio duration}}$ Speed relative to audio — RTF < 1 means faster than real-time Useful
Latency Wall-clock time per utterance End-to-end processing time Useful
VRAM Footprint Peak GPU memory at inference Deployment feasibility Useful
Trainable Params LoRA params / total params Adapter efficiency — lower is more parameter-efficient Useful
Worth reporting in your paper as a table comparing base model vs. LoRA-adapted model. The LoRA trainable param % (typically 0.1–1%) and similar inference RTF demonstrate that fairness improvements come at negligible cost.

Robustness Metrics Perturbation Analysis

Metrics for measuring performance degradation under acoustic perturbations. Your project already includes noise, reverb, and silence perturbations.

MetricFormulaMeasures
WER Under Perturbation $\text{WER}_{\text{perturbed}}$ Absolute performance on degraded audio
Relative WER Increase $\frac{\text{WER}_{\text{perturbed}} - \text{WER}_{\text{clean}}}{\text{WER}_{\text{clean}}}$ Proportional degradation — normalizes for baseline performance
Robustness Gap by Group Relative WER increase per demographic group Whether perturbations disproportionately affect certain groups
Intersection with fairness: A model may have equal WER across groups on clean audio but unequal degradation rates under noise. Reporting the robustness gap by group captures this — it's one of the strongest findings you can make.

Why Not AUC-ROC? Classification Discussion

ASR models perform token-level classification at each output position — predicting a probability distribution over a vocabulary of ~50k tokens. The cross-entropy training loss is literally a classification loss. So why doesn't AUC-ROC apply?

What AUC-ROC Needs vs. What ASR Provides

RequirementAUC-ROCASR Fine-TuningMatch?
Label space Binary or small multi-class ~50,000 token vocabulary No
Independent predictions Required — each sample scored independently Autoregressive: token $t$ depends on tokens $1 \ldots t{-}1$ No
Threshold sweep Core mechanism — varies decision threshold Decoding uses argmax/beam search, no threshold No
Positive examples per class Many per class for stable estimate 50 eval samples → ~300 tokens → most of 50k classes have 0 positives No

Cross-Entropy Already Subsumes AUC's Signal

PropertyCross-EntropyAUC-ROC
Measures ranking of correct class Yes Yes
Measures calibration Yes No
Works with 50k classes Yes Degrades badly
Handles sequential dependence Teacher-forced; valid per-step Assumes independence

Cross-entropy captures everything AUC would tell you (discrimination ability) plus calibration quality (how confident the model is). Switching to AUC would discard information, not add it.

The Sequence-Level Problem

Even with excellent per-token discrimination (hypothetically high AUC per class), transcriptions can fail because:

  • A single high-confidence error at one position cascades into alignment mismatches
  • Beam search amplifies small probability differences
  • Word boundaries and language model context matter more than isolated token ranking

WER captures this. Per-token AUC cannot.

Verdict: AUC-ROC is not applicable here — not because there's no classification happening, but because cross-entropy is already the correct (and strictly superior) classification metric for large-vocabulary token prediction. AUC would be redundant and lossy.
