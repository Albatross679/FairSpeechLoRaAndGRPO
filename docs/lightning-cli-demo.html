USE CASE 2
Deploy: Serve Your Own Models
Write a LitServe server script, then deploy it locally or to the cloud with full control over GPU type, scaling, and cost.
LITSERVE

Write a Server Script

Define four methods — setup, decode_request, predict, encode_response — and LitServe handles the REST API, batching, and GPU management.

Your model loading happens once in setup(). Each request flows through the decode → predict → encode pipeline automatically.

server.py
# server.py import litserve as ls class MyModel(ls.LitAPI): def setup(self, device): from transformers import pipeline self.pipe = pipeline( "text-generation", model="gpt2", device=device ) def decode_request(self, request): return request["prompt"] def predict(self, prompt): return self.pipe( prompt, max_new_tokens=50 )[0]["generated_text"] def encode_response(self, output): return {"text": output} if __name__ == "__main__": api = ls.LitServer(MyModel()) api.run()

From Local to Production
💻

Local Testing

Run locally to validate your server before going to cloud.

lightning deploy api server.py
☁️

Cloud + GPU

Deploy to cloud on any GPU with auto-scaling replicas.

lightning deploy api server.py \
  --cloud \
  --name "text-gen-api" \
  --machine L4 \
  --min-replica 1 \
  --max-replica 4
💰

Spot Instances

Use interruptible GPUs for significant cost savings on batch workloads.

lightning deploy api server.py \
  --cloud \
  --machine A100 \
  --interruptible

FAQ
Common Questions
Answers to frequently asked questions about AI Hub and Deploy.
When should I use AI Hub vs. Deploy?

Use AI Hub when you want a standard model (like LLaMA, Stable Diffusion, Whisper) up and running fast with zero code. Use Deploy when you have a custom model, custom preprocessing, or need fine-grained control over hardware and scaling.

Can I switch from AI Hub to Deploy later?

Yes. AI Hub is great for prototyping. Once you need customization, write a LitServe server.py wrapping the same model and deploy it with full control. The API endpoint format stays the same.

What GPU types are available?

Deploy supports: CPU (various sizes), T4, L4, L40S, A100, H100, H200, and B200 — each available in 1x, 2x, 4x, or 8x configurations. AI Hub uses the GPU type specified by the template author.

How does auto-scaling work?

Set --min-replica and --max-replica when deploying. Lightning automatically scales replicas based on incoming traffic. Scale to zero is possible with min-replica 0.

What are interruptible (spot) instances?

Adding --interruptible uses spot GPU pricing, which can be significantly cheaper. The trade-off is that your instance may be preempted during high-demand periods. Best for batch inference or fault-tolerant workloads.

Do I need to authenticate first?

Inside a Lightning Studio, you're auto-authenticated. From any other environment, run lightning login first. The CLI will open a browser for OAuth.
