# Phase 1.0 Plan 02: Data Loader with Demographic-Stratified Sampling Summary

**PyTorch Dataset and DataLoader with soundfile audio loading, demographic-stratified oversampling sampler, and padded batch collation for GRPO fairness-aware ASR training**

## Accomplishments

- Created ASRFairnessDataset that loads audio via soundfile.read(), validates manifest CSV columns at init, handles missing demographic labels with fillna("") per D-08, and supports both Fair-Speech (ethnicity) and Common Voice (accent) axes
- Created DemographicStratifiedSampler that oversamples small demographic groups to MIN_GROUP_SIZE=50 floor, maintains unlabeled indices for R_acc contribution, and uses numpy RandomState for reproducibility
- Created collate_fn that pads variable-length audio to max-in-batch length with zeros and returns batched tensors with audio_lengths
- Created create_dataloader factory function for one-call setup supporting both training (stratified=True) and evaluation (stratified=False) modes with pin_memory enabled
- Added __main__ smoke test block for quick CLI verification with real data

## Decisions Made

- Used soundfile.read() for audio loading because torchaudio.load() requires torchcodec which is not installed in the environment
- Missing demographic labels stored as empty strings via fillna("") -- included in training pool for R_acc but excluded from stratification per D-08
- Oversampling implemented via repetition-then-truncate pattern: `list(indices) * (floor // len + 1)` then slice to floor size
- collate_fn pads to max-in-batch (not global max) to mitigate OOM risk per threat T-01.0-05
