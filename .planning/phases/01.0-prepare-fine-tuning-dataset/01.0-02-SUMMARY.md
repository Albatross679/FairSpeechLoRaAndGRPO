## Accomplishments

- Created create_dataloader factory providing single entry point for both training (stratified sampling) and evaluation (sequential iteration) with configurable batch size, workers, and epoch length

## Decisions Made
- Used soundfile.read() for audio I/O because torchaudio.load() is broken (requires torchcodec not installed) -- torchaudio.functional.resample() still works for resampling
- Missing demographic labels represented as empty strings (D-08): unlabeled indices added to sampling pool for R_acc contribution but not used for stratification
- Oversampling uses integer repetition then truncation to exact MIN_GROUP_SIZE floor, simpler and more predictable than WeightedRandomSampler
- Audio padding is per-batch (max length in current batch) rather than global max, mitigating OOM risk per threat T-01.0-05
- Added stereo-to-mono conversion as a robustness measure (mean of channels)
