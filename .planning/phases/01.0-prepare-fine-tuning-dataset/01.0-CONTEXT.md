<decisions>
## Implementation Decisions

### Train/Eval Split Strategy
- **D-01:** Fair-Speech 80/20 stratified split by ethnicity, split at the speaker level (speaker_id) to prevent data leakage — ~21K train, ~5.3K eval
- **D-02:** Common Voice 24 test split retained as-is for accent-axis evaluation (no new CV train split needed for Phase 1.0)
- **D-03:** Split must be deterministic (fixed seed) and documented for reproducibility

### Demographic Label Handling
- **D-04:** Parallel axes with no cross-dataset alignment — each dataset keeps its native demographic labels
- **D-05:** Fair-Speech axes: ethnicity (7 categories via existing ETHNICITY_MAP), age (4 bands), L1 language
- **D-06:** Common Voice axes: accent (14 categories via existing ACCENT_MAP), gender, age (decade bins)
- **D-07:** GRPO reward function parameterized by axis — R_fair computed on ethnicity for Fair-Speech batches, accent for Common Voice batches
- **D-08:** Utterances with missing demographic labels excluded from R_fair computation but still contribute to R_acc

### Group Balance & Minimum Sizes
- **D-09:** Stratified oversampling to a minimum floor per group — ensures every training batch contains multiple demographic groups (supporting GRPO-08 stratified batching requirement)
- **D-10:** MIN_GROUP_SIZE=50 threshold retained from existing pipeline for evaluation reporting
- **D-11:** Monitor for small-group memorization by tracking per-group train vs. eval WER divergence during training

### Data Validation
- **D-12:** New validation script following existing validate_*.py pattern (issue collection, summary reporting, exit codes)
- **D-13:** Mandatory checks: demographic label completeness, MIN_GROUP_SIZE per split, speaker/utterance leakage between train and eval, audio file existence, text normalization consistency
- **D-14:** Optional `--plots` flag for matplotlib diagnostic charts (demographic distribution histograms, group size bar charts) — reusable for paper appendix

### Claude's Discretion
- Exact oversampling floor per group (implementation detail for sampler)
- PyTorch Dataset class structure and audio preprocessing pipeline
- Split random seed value
- Validation script output format details

</decisions>

## Existing Code Insights

### Reusable Assets
- `ETHNICITY_MAP` in `scripts/prepare_fairspeech.py` — 7-category ethnicity label standardization, directly reusable
- `ACCENT_MAP` in `scripts/prepare_dataset.py` — 14-category accent label standardization, directly reusable
- `EnglishTextNormalizer` from whisper — already integrated for text normalization consistency
- `IncrementalCSVWriter` in `scripts/run_inference.py` — pattern for fault-tolerant CSV writing

### Established Patterns
- CSV manifests as data contracts between pipeline stages (columns: utterance_id, audio_path, sentence, demographic fields)
- MIN_GROUP_SIZE=50 threshold used consistently across evaluation scripts
- Hardcoded HPC paths as module-level constants — new scripts should follow same pattern
- Validation pattern: collect issues in list, run checks, report summary, exit with code

### Integration Points
- New train/eval manifests feed into Phase 1.1 (LoRA prototype) and Phase 1.2 (standard LoRA baseline) data loaders
- Demographic group labels feed into Phase 2 GRPO reward computation (R_fair per-group WER)
- Oversampling strategy connects to Phase 2 GRPO-08 (demographic-stratified batching)
