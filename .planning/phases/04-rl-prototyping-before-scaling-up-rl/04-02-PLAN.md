**Step 2: Check baseline WER from Plan 01**
Read `outputs/grpo-prototype/lambda_0.0/training_log.json` and extract the mean WER from the last 10 entries. This becomes the `--baseline_wer` argument for the sweep, enabling the WER floor check (D-17) that prevents reward hacking.

```python
import json, numpy as np
log = json.load(open("outputs/grpo-prototype/lambda_0.0/training_log.json"))
baseline_wer = np.mean([e.get("wer/mean", 1.0) for e in log[-10:]])
print(f"Baseline WER from lambda=0.0: {baseline_wer:.4f}")
```

Use this value as `--baseline_wer` in the sweep command.

**Step 3: Run the sweep**
Execute the 3-lambda sweep. This trains 3 models sequentially, each for 200 steps on the same 2K subset. Expect ~2-4 hours total runtime.

```bash
cd /home/zeus/project && \
WANDB_PROJECT=asr-fairness-grpo \
python scripts/training/train_grpo.py \
    --mode sweep \
    --fs_manifest outputs/manifests/fs_train.csv \
    --cv_manifest outputs/manifests/cv_train.csv \
    --output_dir outputs/grpo-prototype \
    --G 2 \
    --max_steps 200 \
    --subset_size 2000 \
    --lr 1e-5 \
    --beta 0.04 \
    --temperature 0.8 \
    --epsilon 0.2 \
    --dr_grpo \
    --baseline_wer <BASELINE_WER_VALUE> \
    --wandb_project asr-fairness-grpo
```

Replace `<BASELINE_WER_VALUE>` with the actual value computed in Step 2.

**Important:** The sweep mode creates a SINGLE data subset and reuses it across all 3 lambdas (see `setup_data` called once in `run_sweep`). This is correct -- same data ensures fair comparison. However, it loads a FRESH model for each lambda value (see `setup_model()` inside the loop).

**Note:** The sweep will overwrite `outputs/grpo-prototype/lambda_0.0/` from Plan 01. This is expected behavior -- the sweep re-trains lambda=0.0 with the `--baseline_wer` floor check active. The Plan 01 adapter was for signal validation only.

**If a specific lambda fails:**
- The sweep continues to the next lambda (each is independent after model load)
- If lambda=0.3 or 0.7 shows NaN loss, the fairness reward may have numerical issues -- check if per-group WER computation is getting empty groups
- If VRAM increases compared to Plan 01, the fairness rolling window may be accumulating tensors -- check reward.py for leaks

**Step 2: Build sweep comparison JSON**

Aggregate the evaluation results into a single comparison file:

```python
import json, os, glob

comparison = {"lambdas": {}, "metadata": {
    "G": 2, "max_steps": 200, "subset_size": 2000, "mode": "prototype"
}}

for lambda_val in [0.0, 0.3, 0.7]:
    ldir = f"outputs/grpo-prototype/eval/lambda_{lambda_val}"
    entry = {"lambda": lambda_val}

    # Training metrics
    summary_path = f"outputs/grpo-prototype/lambda_{lambda_val}/training_summary.json"
    if os.path.exists(summary_path):
        entry["training"] = json.load(open(summary_path))

    # Training log (last 10 entries for trend)
    log_path = f"outputs/grpo-prototype/lambda_{lambda_val}/training_log.json"
    if os.path.exists(log_path):
        log = json.load(open(log_path))
        if log:
            entry["signal"] = {
                "frac_zero_std_last10": float(sum(e.get("reward/frac_zero_std", 1.0) for e in log[-10:]) / 10),
                "advantage_std_last10": float(sum(e.get("advantage/std", 0.0) for e in log[-10:]) / 10),
            }

    # Eval metrics (if evaluation ran)
    for dataset in ["fs", "cv"]:
        metrics_path = os.path.join(ldir, f"{dataset}_fairness_metrics.json")
        if os.path.exists(metrics_path):
            entry[f"{dataset}_eval"] = json.load(open(metrics_path))

    comparison["lambdas"][str(lambda_val)] = entry

# Save
os.makedirs("outputs/grpo-prototype/eval", exist_ok=True)
with open("outputs/grpo-prototype/eval/sweep_comparison.json", "w") as f:
    json.dump(comparison, f, indent=2, default=str)

print("Sweep comparison saved")
```
