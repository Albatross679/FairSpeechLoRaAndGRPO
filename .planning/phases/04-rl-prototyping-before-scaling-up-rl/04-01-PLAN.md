<objective>
Run the GRPO prototype with lambda=0.0 (pure accuracy) to validate the entire training pipeline works end-to-end on a T4 GPU. This resolves four open questions: (1) does Qwen3-ASR support temperature sampling, (2) what is the actual VRAM profile for G=2 GRPO, (3) does the learning rate produce stable training, and (4) does the reward produce nonzero advantage variance.

Purpose: Gate for Plan 02 (lambda sweep). If the prototype fails (OOM, no training signal, generation errors), the sweep would waste GPU time.
Output: training_log.json and training_summary.json confirming GRPO viability, plus a trained lambda=0.0 adapter.
</objective>

**If the script hits an error:**

- **OOM**: Note peak VRAM from error output. If close to 15GB, try reducing max_new_tokens to 64: add `--max_new_tokens 64`. If still OOM, this blocks Phase 4 -- document in experiment log and escalate.

- **Generation error** (e.g., `do_sample` not supported): This means Qwen3-ASR does not support temperature sampling. Document the error. The fallback is to modify generate_candidates() in train_grpo.py to use `do_sample=False` (greedy) and rely on dropout for diversity, but this is a significant risk to training signal.

- **Import error**: Check that sys.path insert in train_grpo.py correctly resolves. Run from project root: `cd /home/zeus/project`.

- **Data error**: Verify manifest CSV has columns: audio_path, sentence, plus demographic columns (ethnicity or accent).

**Step 1: Analyze training signal**

Run a Python analysis to validate training signal quality per D-21:

```python
import json
import numpy as np

log = json.load(open("outputs/grpo-prototype/lambda_0.0/training_log.json"))

# Signal validation (D-21)
last_10 = log[-10:]
first_10 = log[:10]

avg_zero_std_last = np.mean([e.get("reward/frac_zero_std", 1.0) for e in last_10])
avg_adv_std_last = np.mean([e.get("advantage/std", 0.0) for e in last_10])
avg_loss_first = np.mean([e.get("loss", 0.0) for e in first_10])
avg_loss_last = np.mean([e.get("loss", 0.0) for e in last_10])
avg_reward_first = np.mean([e.get("reward/mean", 0.0) for e in first_10])
avg_reward_last = np.mean([e.get("reward/mean", 0.0) for e in last_10])
avg_wer_first = np.mean([e.get("wer/mean", 1.0) for e in first_10])
avg_wer_last = np.mean([e.get("wer/mean", 1.0) for e in last_10])
avg_clip_frac = np.mean([e.get("clip_frac", 0.0) for e in last_10])

print("=== GRPO SIGNAL VALIDATION ===")
print(f"frac_zero_std (last 10): {avg_zero_std_last:.3f}  {'PASS' if avg_zero_std_last < 0.5 else 'FAIL'}")
print(f"advantage_std (last 10): {avg_adv_std_last:.4f}  {'PASS' if avg_adv_std_last > 0.01 else 'FAIL'}")
print(f"Loss trend: {avg_loss_first:.4f} -> {avg_loss_last:.4f}  {'PASS' if avg_loss_last < avg_loss_first else 'FLAT/WARN'}")
print(f"Reward trend: {avg_reward_first:.4f} -> {avg_reward_last:.4f}")
print(f"WER trend: {avg_wer_first:.4f} -> {avg_wer_last:.4f}")
print(f"Clip fraction (last 10): {avg_clip_frac:.3f}")

# Per-group WER if available
group_keys = [k for k in last_10[-1].keys() if k.startswith("wer/group_")]
if group_keys:
    print("\nPer-group WER (last entry):")
    for k in sorted(group_keys):
        print(f"  {k}: {last_10[-1][k]:.4f}")

# Gate check
signal_ok = avg_zero_std_last < 0.5 and avg_adv_std_last > 0.01
print(f"\n=== GATE: {'PASS - proceed to sweep' if signal_ok else 'FAIL - training signal too weak'} ===")
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Denial of Service | GPU OOM during G=2 generation | mitigate | VRAMMonitor checks every 10 steps, budget 13GB; sequential generation with cache clearing; fallback: reduce max_new_tokens to 64 |
| T-04-02 | Information Disclosure | W&B logs training metrics publicly | accept | Research project, no sensitive data; metrics are WER/reward values |
| T-04-03 | Tampering | Reward hacking (WER increases while reward increases) | mitigate | WER floor check in RewardComputer (D-17); per-group WER logged for manual inspection |
| T-04-04 | Denial of Service | generate() hangs or infinite loop | mitigate | max_new_tokens=128 caps generation length; timeout would require external wrapper |
