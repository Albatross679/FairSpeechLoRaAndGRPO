The script has two modes: `validate` and `train`.

**1. Module-level constants:**
```python
MODEL_ID = "Qwen/Qwen3-ASR-1.7B"
SEED = 42
DEFAULT_OUTPUT_DIR = "outputs/standard-lora"
DEFAULT_HP_SWEEP_DIR = "outputs/hp-sweep"
VRAM_BUDGET_GB = 14.0
```

**2. Reuse existing patterns (import or copy from phase2_hp_sweep.py):**
- `load_model_and_processor()` — load via qwen-asr wrapper
- `patch_outer_forward()` — forward delegation to thinker
- `apply_lora(model, rank, alpha, dropout, target_mlp)` — LoRA application with encoder freezing
- `print_gpu_memory(label)` — VRAM profiling
- `create_stratified_subset()` — **MUST be modified from phase2_hp_sweep.py's proportional split to enforce equal FS/CV split per D-02**: set `fs_n = subset_size // 2` and `cv_n = subset_size - fs_n` instead of proportional `fs_fraction = len(fs_df) / total`

**3. Validate mode (`--mode validate`):**
- Reads `top3_configs.json` from `--hp_sweep_dir`
- For each of the top 2-3 configs:
  a. Load fresh model, apply LoRA with that config's HPs
  b. Create 1000-sample stratified subset with equal FS/CV split (500 each, per D-02)
  c. Create 90/10 speaker-disjoint train/eval split (same speaker-disjoint logic as train mode per D-03)
  d. Train for 300 steps with HF Trainer (30% of estimated ~1000 full steps)
  e. Evaluate on eval split, record eval_loss
  f. Log to W&B under project "asr-fairness-lora", run name "validate-config-{i}"
  g. Aggressive GPU cleanup between configs (del model, torch.cuda.empty_cache, gc.collect)
- After all configs validated:
  a. Print ranking comparison: HP sweep rank vs validation rank
  b. Save `validation_results.json` with all configs + validation losses
  c. If ranking preserved (same top-1), print "GATE PASSED"
  d. If ranking flipped, print "GATE WARNING" with recommendation to increase subset size
  e. Save `locked_config.json` with the best-performing validated config

**4. Train mode (`--mode train`):**
- Reads `locked_config.json` from `--output_dir` (produced by validate mode)
- Pre-flight checklist (per ML prototype Phase 4):
  a. Verify locked_config.json exists
  b. Run nvidia-smi check (print GPU name, total VRAM, current usage)
  c. Verify manifests exist
  d. Print estimated wall-clock time (~30-60 min for 1K steps on 1-2K samples)
- Create full 1-2K sample stratified subset (equal split FS/CV per D-02):
  a. ~500-1000 from Fair-Speech, ~500-1000 from Common Voice (fs_n = subset_size // 2)
  b. **Speaker-disjoint eval set (D-03 / Pitfall #9):**
     - Check if manifests contain a `speaker_id` or equivalent column (e.g., `client_id` in Common Voice, discoverable from Fair-Speech metadata)
     - If speaker_id available: group all utterances by speaker_id, assign entire speakers (with all their utterances) to either train or eval — never split a speaker across both. Use stratified group split: `sklearn.model_selection.GroupShuffleSplit(n_splits=1, test_size=0.1, random_state=SEED)` with groups=speaker_ids
     - If speaker_id NOT available: fall back to random stratified split by demographic group, log a WARNING that speaker-disjoint cannot be guaranteed, and document this limitation
     - After splitting, ASSERT `len(set(train_speaker_ids) & set(eval_speaker_ids)) == 0` when speaker_ids are available
  c. Save the train and eval subset CSVs to output_dir for reproducibility and so Plan 02 evaluation can use the exact eval subset
- Train with HF Trainer:
  a. Load locked HP config
  b. `max_steps` = full training (800-1200 steps, Claude decides based on subset size and batch size)
  c. `per_device_train_batch_size=2`
  d. `gradient_accumulation_steps=2` (effective batch 4)
  e. `bf16=True`
  f. `eval_strategy="steps"`, `eval_steps=100`
  g. `save_strategy="steps"`, `save_steps=200`
  h. `logging_steps=10`
  i. `report_to="wandb"`, W&B project "asr-fairness-lora", run name "standard-lora-final"
  j. `remove_unused_columns=False`
  k. `dataloader_pin_memory=False`
  l. Warmup: 5% of total steps
  m. LR scheduler: cosine
- VRAM monitoring: `print_gpu_memory()` at model load, after PEFT wrap, every 200 steps, end of training
- After training:
  a. Save adapter: `model.save_pretrained(output_dir/adapter)`
  b. Save processor: `processor.save_pretrained(output_dir/adapter)`
  c. Save training config JSON with all HPs, step count, final loss
  d. Print final loss, VRAM peak, training time
  e. Verify adapter files exist (adapter_model.safetensors, adapter_config.json)
